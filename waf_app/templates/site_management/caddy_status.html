Caddy Setup Instructions

1 Install Caddy Server

For Debian/Ubuntu systems: 

curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
sudo apt update
sudo apt install caddy

2 Enable Caddy Admin API

Create or edit your Caddyfile at /etc/caddy/Caddyfile: 

{
    admin localhost:2019
    # Optional: Enable admin API on all interfaces (use with caution)
    # admin 0.0.0.0:2019
}

# Your sites will be configured here automatically by the WAF

3 Start and Enable Caddy 

sudo systemctl enable caddy
sudo systemctl start caddy
sudo systemctl status caddy

4 Configure WAF Settings

Add to your Django settings.py: 

# Caddy Configuration
CADDY_API_URL = 'http://localhost:2019'  # Or your Caddy admin API URL
CADDY_BASE_PATH = '/etc/caddy'  # Path where Caddy stores configs

✓ Verify Connection

Once configured, refresh this page to verify the connection to Caddy.

Important Notes

  • The Caddy Admin API must be accessible from this server
  • Ensure firewall rules allow communication on port 2019
  • For production, consider securing the Admin API with authentication
  • Caddy will automatically obtain SSL certificates from Let's Encrypt when configured
